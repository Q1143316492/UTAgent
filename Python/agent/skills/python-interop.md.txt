## 三层决策树

```
用户任务
  → execPython（持久 ScriptEnv 命名空间，预置 unity + CS）
      L1: import unity          — 高频动词（截图、层级、Scene View…）
      L2: unity.list_editor_namespaces / get_type_details — 发现 API（已过滤）
      L3: from unity_bind import CS — Editor 开放动态 CLR
```

**禁止**：首步 `help(unity)` / `dir(unity)` / 无过滤 `list_namespaces()` / 读 `Assets/**/*.cs` / `import clr`。

**是否新增 L1 动词**：须对照现有 `unity` 动词清单再立项，本模块不擅自扩展。

## 观察回灌（流程关键）

| 写法 | Agent tool 结果里能否看到 |
|------|---------------------------|
| `unity.get_hierarchy()` / `find_object` / `find_objects` / `get_type_details` | ✅ 自动 echo 到 stdout |
| `print(...)` | ✅ stdout |
| `unity.log(...)` | ❌ 仅 Unity Console，**不能**当侦察依据 |

- 侦察脚本末行用 `print` 或让 L1 动词作末行表达式；**不要**连跑多步「空 stdout 侦察」
- `find_objects` 返回 **dict**，用 `result["count"]`，不是 `len(result)`

## 任务类型

| 用户任务 | 做法 |
|----------|------|
| 拼/改 Editor UI、预制体、布局 | **先** `loadSkill("editor-ui")`，再单步 execPython |
| **UI 排查**（点不了、错位、查脚本） | **先** `loadSkill("editor-ui-debug")`，再 golden-path 单步 exec |
| 不确定场景有无 Canvas | 最多 **1 步** `find_object("Canvas")` 再创建 |
| 上一步失败 | patch / destroy_all，禁止盲目重建（见下） |

## Edit Mode 默认假设

Agent **默认在 Edit Mode**（`Application.isPlaying == false`），除非用户明确说在 Play Mode。

不确定时可在 exec 内读取：

```python
from unity_bind import CS
is_playing = CS.UnityEngine.Application.isPlaying
```

**Rule**：用户未声明 Play Mode 时，一律按 Edit Mode 安全 API 写代码。

## UI 任务必读（Edit Mode）

Editor UI 创建/排查时 **优先遵守** 下列规则（硬验收也覆盖这些）：

| 陷阱 | UI 场景 | ✅ 正确做法 |
|------|---------|------------|
| `Object.Destroy` | 删测试 Btn / Wnd | `unity.destroy_object(name)` |
| 创建前同名残留 | 重复堆叠 `Btn*` / `Wnd*` | 脚本开头 `unity.prepare_scene_object(目标名)` |
| `GameObject.Find(name)` | 查 **inactive** 的 `WndLogin` 等 | `unity.find_objects(name)`（含非激活） |
| `Instantiate(prefab)` | 把 UI prefab 放进 Canvas | L3：`CS.UnityEditor.PrefabUtility.InstantiatePrefab(asset)` |
| `SetActive` 后等 `Awake`/`Start` | 新建 UI 以为逻辑已跑 | Edit Mode **不调用** Awake/Start；手动设字段或调初始化 |
| 第二个 Canvas | 创建 UI 根 | 场景已有 `Canvas` 时 **复用**，勿再 `new GameObject("Canvas")` |
| Image + TMP 同 GO | 按钮结构 | 根 **Image+Button**，子 `TxtLabel` 挂 **TMP**（互斥） |
| `AddComponent(RectTransform)` 返回 None | UI 子节点拉满父级 | 先 `AddComponent(Image/TMP)` 再 **`GetComponent(RectTransform)`** 设 anchor；**禁止**对已有 `Transform` 的 GO `AddComponent(RectTransform)` |

**创建前**：`unity.prepare_scene_object(目标根名)`。**失败修补**：读 tool result 的 `recommended_action`；`patch_existing_only` 时禁止整段重建。

**Graphic 互斥**：同一 GameObject 不能同时挂 `UnityEngine.UI.Image` 与 `TextMeshProUGUI`。

## Edit Mode 安全替换表

除非已确认 `Application.isPlaying == true`，否则用右列替代左列：

| ❌ 运行期 / 危险 API | ✅ Edit Mode 安全替代 | 说明 |
|---------------------|----------------------|------|
| `renderer.material` | `renderer.sharedMaterial` | `.material` 会实例化副本，Edit Mode 易泄漏（**UI 任务通常不涉及**） |
| `renderer.materials` | `renderer.sharedMaterials` | 同上 |
| `meshFilter.mesh` | `meshFilter.sharedMesh` | 同上 |
| `Object.Destroy(obj)` | `unity.destroy_object(name)` 或 L3 `DestroyImmediate` | `Destroy` 延迟执行，Edit Mode 常删不掉 |
| `StartCoroutine(...)` | 不使用；或 L3 `EditorApplication.update` | 协程在 Edit Mode 不跑 |
| `Instantiate(prefab)`（运行期_spawn） | L3 `PrefabUtility.InstantiatePrefab` | Edit Mode 保持 prefab 关联 |
| `SetActive` 后依赖 `Awake`/`Start` | 手动初始化组件状态 | 生命周期在 Edit Mode 不触发 |

**读材质/网格属性**：只读时用 `sharedMaterial` / `sharedMesh`；勿用 `.material` / `.mesh`。**改**某对象专属材质：Play Mode 用 `.material`；Edit Mode 须显式 `Instantiate(sharedMaterial)` 再赋回 `sharedMaterial`（或接受改 shared 影响所有引用者）。UI 配色任务优先走 `editor-ui` skill，勿在 exec 里乱改 Renderer 材质。

**销毁**：L1 优先 `unity.destroy_object` / `unity.prepare_scene_object` / `unity.destroy_all_objects`；L3 等价 `CS.UnityEngine.Object.DestroyImmediate(go)`。

## unity 4 子模块速查

`unity` 按域拆 4 子模块，按域 import 更清晰；`unity.<verb>` 顶层路径仍可用（兼容层）。

| 子模块 | 核心动词 |
|--------|----------|
| `unity.scene_view` | `get_hierarchy` / `find_objects` / `destroy_object` / `destroy_all_objects` / `prepare_scene_object` / `create_primitive` / `save_scene` / `select_game_object` / `focus_scene_view_on` / `scene_view_zoom/pan/orbit` / `get/set_scene_view_camera` |
| `unity.screenshot` | **`capture`**（`view` / `name` 裁切 / `save_to_file`）；旧名 `capture_screenshot` / `capture_scene_view` 仅转发 |
| `unity.inspect` | `list_namespaces` / `list_editor_namespaces` / `list_types_in_namespace` / `get_type_details` |
| `unity.console` | `get_logs` / `get_log_summary` / `log` / `log_warning` / `log_error` |

```python
from unity.scene_view import get_hierarchy, prepare_scene_object, find_objects
from unity.inspect import get_type_details
```

**Legacy deprecated**（`unity.*` shim 仍可调，指引替代）：
- `create_cube` → `create_primitive("Cube", name, position)`
- `find_object` → `find_objects` 或 `scene_view.get_hierarchy`
- `get/set_position/rotation/scale` / `move_object` / `rotate_object` / `look_at` → L3 `CS.UnityEngine.Transform`

## L1 — `import unity`（动词优先）

- 截图：`unity.screenshot.capture(view="scene")`（勿 print 含 base64 的返回值）
- **Cursor / 能看图的模型**：`capture(..., save_to_file=True)` 或 CLI `utagent screenshot [--name UI节点]` → 用返回的 `path` 做目检
- 按节点裁切：`capture(view="scene", name="WndStardewHud", save_to_file=True)`（含 World Space Canvas）
- **DeepSeek / 纯文本模型**：验收用 `find_objects` + `print`，不要为验收调截图；默认 `__image` 会被剥离不送入 LLM
- 层级：`unity.find_object` / `unity.find_objects` / `unity.get_hierarchy`（`components` 含 `shortName` + `fullName`）
- 销毁：`unity.destroy_object` / `unity.prepare_scene_object` / `unity.destroy_all_objects`
- 镜头：`unity.scene_view_*` / `unity.focus_scene_view_on`

有对应 L1 动词时 **必须** 用 L1，不要跳到 L3。

## L2 — 过滤后自省

```python
import unity
unity.get_type_details("UnityEngine.UI.Button,TMPro.TextMeshProUGUI")
```

L2 仍经 `UnityBindWhitelist` 过滤；**不代表** L3 exec 受同一白名单限制。

## L3 — `from unity_bind import CS`（动词不够时）

**Editor exec 对已加载程序集动态反射**，无运行时白名单 Filter。

- `CS.GameCore.WindowBase` 等业务类型可用（程序集已在 Unity 域）
- `CS.UnityEditor.MonoScript` / `AssetDatabase` / `PrefabUtility` 等长尾 Editor API 可用
- **找资源**用 `AssetDatabase.FindAssets` / `LoadAssetAtPath`；禁止 `os.walk` / `Path.rglob` / 递归 `glob` 扫 `Assets`
- 场景已有 `Canvas` 时 **不要** 再新建第二个 Canvas
- Edit Mode 实例化 prefab：`CS.UnityEditor.PrefabUtility.InstantiatePrefab(asset)`
- 创建类脚本末行用 `print(...)` / `unity.save_scene()`
- Editor **创建/改** UI 走 L1+L3 + `loadSkill("editor-ui")`；**排查**走 `editor-ui-debug`

### pythonnet 陷阱（L3）

- **extension method**（如部分 LINQ 扩展）在 pythonnet 下常失败；改用显式静态调用或 golden-path 已验证写法
- **`out` / `ref`**：pythonnet 与 C# 签名不匹配时易报错；优先改脚本结构，勿 `import clr` 手搓
- **泛型实例化**（`List[T]` 等）：可能失败；优先用 golden-path 或非泛型 API
- **重载解析**：歧义时指定参数类型或换等价 API
- 查 **脚本路径 / Rect / Button 状态**：用 `loadSkill("editor-ui-debug")` 单步 golden-path，勿多步空 `get_hierarchy`
- 不要 `import clr`；程序集预载由 `unity_bind` 内部 `ensure_clr` 完成
- 域重载后若类型解析失败，提示用户 `utagent init` 或重开 exec 会话
